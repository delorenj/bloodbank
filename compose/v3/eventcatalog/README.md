# EventCatalog (event discovery UI)

EventCatalog is the human- and agent-facing discovery surface for 33GOD v3
events, commands, services, and domains. It renders content generated by
**Holyfields** — AsyncAPI documents, CloudEvents schemas, command envelopes,
and per-service metadata.

EventCatalog is **not** a source of truth. It consumes. The generator lives
in Holyfields (tracked under HOLYF-2; see
`docs/architecture/v3-holyfields-contract-work.md` once V3-010 lands).

## Local mount path convention

The sandbox expects generated catalog content at:

```
compose/v3/eventcatalog/site/
```

on the host; the `eventcatalog` service in `../docker-compose.yml` mounts
that path read-only at `/app/site` inside the container. The directory is
intentionally empty in this scaffold. When Holyfields generation lands,
its CI (or a local operator command) will populate `site/` with:

- `events/` — per-event markdown + JSON schema references
- `commands/` — per-command markdown + JSON schema references
- `services/` — per-service AsyncAPI docs and metadata
- `domains/` — cross-cutting domain descriptions

Until then, the EventCatalog container will start but show an empty site.
That is expected.

Runtime-specific artifacts (build scripts, seed content, export snapshots)
will be added to this directory by later tickets without changing the v3
compose root.
